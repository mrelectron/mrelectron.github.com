RTorrent Installation

May 15th, 2012

This post describes the process necessary to install and configure rtorrent on a remote machine running Ubuntu or Mint. I need to document this process for my own use, because it took me longer to setup that it should have and I don’t want to have to repeat that experience. I want to thank Randy for his init.d script.

Assumptions

You are familiar with basic Linux concepts such as sudo, home directory, opening terminal windows and entering shell commands.

Motivation

For a while now I have been running Windows + utorrent. Recently we installed a 100Mb/40Mb fibre link, so I thought I would setup a spare netbook as an always on seedbox. Then I would have not to leave my desktop running overnight. I also wanted a fire and forget solution. One that pickups .torrent files from a watch directory, downloads, seeds, and finally stops seeding once a predefined ratio is reached.

Prerequisites

Fresh install of Linux Mint on the netbook. That will be our starting point.

Step 1 - Configure Router

I had this step last, but unless you know how to do this, then there is no point in doing the other steps. A lot of GUI torrent clients have uPnP support,which means that the torrent client speaks to your router and says please ma’am can you open a port for me. But I’m not able to say if rtorrent has uPnP support. In any case I prefer to open my router ports manually. Now as router configuration is model specific, all I can tell you for the purposes of this “how to” is you need to forward port range 60125-64125 for TCP traffic to the IP address of the machine on which rtorrent is running.

Workflow

The way in which I use this setup. I have a access to the netbook watch directory from my windows machine via a Samba share. When I see something on the web I want to download, I download the .torrent file to my windows machine and copy it to the netbook watch folder. rtorrent will pick it up from the watch folder, download the file, and then will continues to seed the file until the seed ratio set in the .rtorrent.rc file is met.

Subsequently at my leisure I will pickup the downloaded file and copy it to wherever I want its final resting place to be.

I could easily make some improvements to this workflow, for example once the seed ratio has been met, automagically move the file to its final resting place without my intervention. RTorrent does have excellent functionality to run different processes at different times.

Anytime you actually want to use rtorrent interactively, just run screen -dr, which will detach the screen that rtorrent is running under. Just remember though once you are using it interactively and you quit out of it you will have to start it again via the daemon.
