RTorrent Install, Config, AutoStart

May 15th, 2012

This post describes the process necessary to install and configure rtorrent on a remote machine running Ubuntu or Mint. I need to document this process for my own use, because it took me longer to setup that it should have and I don’t want to have to repeat that experience. I want to thank Randy for his init.d script.

Assumptions

You are familiar with basic Linux concepts such as sudo, home directory, opening terminal windows and entering shell commands.

Motivation

For a while now I have been running Windows + utorrent. Recently we installed a 100Mb/40Mb fibre link, so I thought I would setup a spare netbook as an always switched on seedbox. Then I would have not to leave my desktop running overnight, as it is a power hungry beast.

I also wanted a fire and forget torrent solution. One that pickups .torrent files from a watch directory, download them and then seed them to a certain ratio.

Prequisites

Fresh install of Linux Mint on the emachine. That will be our starting point.

Step 1 - Configure Router

I had this step last, but unless you know how to do this, then there is no point in doing the other steps. A lot of GUI torrent clients have uPNP support,which means that the torrent client speaks to your router and says please ma’am can you open a port for me. But I’m not able to say if rtorrent has uPNP support, so I open ports on my router manually. As router configuration is model specific, all I can tell you is you need to forward port range 60125-64125 for TCP traffic to the IP address of the machine on which rtorrent is running.
